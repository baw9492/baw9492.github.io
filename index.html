《How to Find a Place Suitable for “Guitar Practice”:Purpose-oriented Geographic Entity Retrievalby Using Online Review Graph Analysis》总结

文章内容

大多数现有地图搜索引擎只接受地点名称或地点类型。因此，当搜索者想要找到一个适合“吉他练习”的地点时，他们必须输入一个地点类型，例如“音乐工作室”。要进行这样的搜索，需要先验知识（即知道音乐工作室适合弹吉他）。作者提出了一种 以目的作为搜索索引，从而找出能实现目的的地点的搜索方法。这种搜索方法允许用户在没有事先了解地点信息的情况下成功地进行搜索。

作者进行的相关工作:

  • 地点信息检索 ：利用了 Pat 等人开发的一个地理信息检索系统、Shoji 等人提出的基于Word2vec的“location2vec”算法、Bauer等人提出的一种搜索实体商店的方法，Kato等人提出的相似地点查询算法
  • 目的扩展 ：借鉴了 Pothiratanachaikul等人提出的一种从社区问答（cQA）站点中提取可实现相同目的的备选方案的方法、 Jiwoon和Wang等人提出的计算目的相似度的方法。
  • 地点推荐 ：旨在找到一个可以实现用户目的的地方。利用了Kurashima等人提出的一种提取地点特征的方法

作者提出的方法：

因为并非所有可以练习吉他的地点都有这样的在线评论：“我在这里练习吉他”。因此，基于简单匹配的传统信息检索方法无法充分利用这些评论。 为了扩展搜索结果，作者在基于图的算法中采用了三个假设：

  1. 相互循环推导（可以达到相同目的的地方彼此相似）
  2. 通过地点类型推导（相同的目的可以在类型相似的地方实现）
  3. 通过词义推导（相似的目的可以在相同的地方实现）

为了实现基于以上假设推导的目的扩展 并 对结果进行排名，作者提出了以下解决方法

  1. 为相互循环创建二分图
  2. 基于地点类型扩展的地点相似度计算
  3. 为扩展目的计算单词相似度
  4. 通过重启随机游走算法对地点进行排名

作者进行的实验:

收集数据

通过 Google Maps 的 Places API 收集地点评论数据和地点元数据。

实施

作者使用了名为 mecab-ipadic-NEologd 的词典，其中包括社交媒体服务中常用的新词 为了计算地点之间的相似度，作者使用了类别标签

比较方法

为了分析三个假设的有效性，作者准备了以下五种方法:

  • All(H1, H2, H3) 是考虑所有假设的方法
  • Place Only(H1, H2) 是一种仅考虑地点类型相似性的变体方法
  • Word Only(H1, H3) 是另一种只考虑单词语义相似度的变体方法
  • No Expansion(H1 only) 是一种简单的方法，不考虑地点和词的相似性
  • Baseline 是一种传统的搜索算法，它只查找评论直接包含所有查询词的地方。

结果

作者评估了每个方法和每个查询的精度和排名评估以及实际输出。总体结果是，所有提出的其他方法都实现了比Baseline更高的精度。 对于所有查询的平均结果，Place Only 的效果最好。但是，每个扩展都有不同的优势。

  • 简单了解了"重启随机游走算法"：

随机游走算法的基本思想是，从一个或一系列顶点开始遍历一张图。在任意一个顶点，遍历者将以概率1-a游走到这个顶点的邻居顶点，以概率a随机跳跃到图中的任何一个顶点，称a为跳转发生概率，每次游走后得出一个概率分布，该概率分布刻画了图中每一个顶点被访问到的概率。用这个概率分布作为下一次游走的输入并反复迭代这一过程。当满足一定前提条件时，这个概率分布会趋于收敛。收敛后，即可以得到一个平稳的概率分布。随机游走模型广泛应用于数据挖掘和互联网领域，PageRan算法可以看作是随机游走模型的一个实例。

重启随机游走算法是在随机游走算法的基础的改进。从图中的某一个节点出发，每一步面临两个选择，随机选择相邻节点，或者返回开始节点。算法包含一个参数a为重启概率，1-a表示移动到相邻节点的概率，经过迭代到达平稳，平稳后得到的概率分布可被看作是受开始节点影响的分布。重启随机游走可以捕捉两个节点之间多方面的关系，捕捉图的整体结构信息。

  • 简单了解了word2vec算法：

设有

f(x,y) f(x,y)f(x,y)

f为自然语言处理(NLP)中经常出现的【语言模型】，这个模型的目的，就是判断单词x和单词y这个样本，是否符合自然语言的法则，更通俗点说就是：词语x和词语y放在一起，是不是人话。

Word2vec 正是来源于这个思想，但它的最终目的，不是要把 f 训练得多么完美，而是只关心模型训练完后的副产物——模型参数，并利用这些参数找到词x与词y之间的关系。
